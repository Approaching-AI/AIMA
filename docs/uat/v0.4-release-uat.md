## 执行原则

遵循 `CLAUDE.md` "ALL COLLECT, THEN ANALYZE" 流程：

1. 本机交叉编译所有目标二进制
2. `scp` 到所有目标设备
3. 在所有相关设备上**并行**跑同一组测试命令
4. 等齐所有结果（不可达 = UNREACHABLE，不得跳过）
5. 用结果矩阵做统一分析
6. 一次性修改，避免"按下葫芦浮起瓢"
7. 每项 UAT 必须保留可追溯证据（日志、DB 快照、截图 GIF）到 `artifacts/uat/v0.4/<uat-id>/`

  4. Edge：benchmark 失败（OOM 或 crash）

  - `rejection_reason` 在 central advisory 记录可见
  - 同一 advisory 不会再作为 `pending` 被下发
  - edge 本地 exploration_plans 里该任务被标记 `rejected`
